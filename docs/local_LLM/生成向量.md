# 生成向量

## 建立向量資料庫

`ts-node src/ingest.ts`

## 重建向量資料庫

### 刪掉原本的 collection（最乾淨）

Chroma Server  
`curl -X DELETE http://localhost:8000/api/v2/collections/你的collection名稱
`

重新跑  
`ts-node src/ingest.ts`

### 重建新 collection 名稱（避免衝突）

`const COLLECTION = "company_rag_v2";`

> 好處：
>
> - 舊資料還留著可比對
> - 測試比較安全
