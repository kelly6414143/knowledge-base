# 🖥 一、Windows 多桌面基本操作

## 🔹 開啟工作檢視

```
Win + Tab
```

上方會看到目前的桌面列表。

---

## 🔹 建立新桌面

```
Win + Ctrl + D
```

會直接建立一個新的虛擬桌面。

---

## 🔹 切換桌面

```
Win + Ctrl + →
Win + Ctrl + ←
```

超順，比滑鼠點快很多。

---

## 🔹 關閉當前桌面

```
Win + Ctrl + F4
```

⚠ 不會關掉程式
會把該桌面的視窗移到前一個桌面。

---

# 🎛 二、實戰管理技巧（這才是重點）

## 1️⃣ 幫桌面命名

按 `Win + Tab`
在桌面名稱上點一下就可以改名。

例如：

- 💻 Frontend Dev
- 🧠 LLM Lab
- 📄 Docs / iPAS
- 📈 投資分析

你是重度工程流的人，我會強烈建議一定要命名。

---

## 2️⃣ 固定某個 App 出現在所有桌面

在 `Win + Tab` → 右鍵某個視窗 →
選：

- 「在所有桌面顯示此視窗」

例如：

- LINE
- Spotify
- 記錄工具

這樣就不用每個桌面都開一次。

---

## 3️⃣ 拖曳視窗到其他桌面

`Win + Tab`
直接把視窗拖到別的桌面。

這個整理專案時超好用。

---

# 🧠 三、進階管理策略（工程師版）

你現在常做的事大概包含：

- React + TS 開發
- RAG + OCR + 向量資料庫
- Cursor / Prompt 調教
- 學 iPAS
- 投資分析

我會建議這樣分：

---

## 🖥 桌面1：專案開發

- IDE（Cursor / VSCode）
- Chrome DevTools
- Postman

👉 專心寫 code

---

## 🧠 桌面2：LLM / RAG

- Terminal
- Ollama
- ChromaDB
- 測試 prompt

👉 測試模型，不干擾主開發

---

## 📄 桌面3：文件整理

- Markdown
- prototype
- OCR 檢查

👉 純閱讀 + 結構整理

---

## 📈 桌面4：生活 & 投資

- 股票
- 財報
- 新聞

👉 不要讓它干擾工作桌面 😆

---

# 🔥 四、多桌面常見錯誤

❌ 每個桌面都開一堆 Chrome
❌ 不命名
❌ 不固定通訊工具
❌ 不用快捷鍵（只用滑鼠）

---

# 🚀 五、效率升級技巧

如果你用雙螢幕：

👉 左螢幕固定桌面1
👉 右螢幕切換桌面2/3

會變成「分層工作流」。
