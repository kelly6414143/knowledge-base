# LLM 名詞解釋

## Ollama 是什麼

- 本地運行模型的工具
- 幫你下載、管理模型

指令

```=bash
ollama run qwen2.5:7b
```

### Qwen2.5 是什麼

> 阿里巴巴開源的 大語言模型（LLM）系列，與 GPT、Llama、Gemini 類似。

- 支援多語言（中文特別強）
- 能寫程式、寫文件、聊天
- 可本地化運行（不需要雲端）

### 7B 是什麼意思

> 表示它的 參數量是 7 Billion（70 億）。
> 參數量就像：
>
> > 大腦的「神經元」  
> > 越大能力越強、但吃更多記憶體

| 參數量 | 需要記憶體   | 能力               |
| ------ | ------------ | ------------------ |
| 3B     | 4GB ↑        | 基礎問答           |
| **7B** | **8GB–16GB** | 一般好用、性價比高 |
| 14B    | 16GB–24GB    | 能力明顯更強       |
| 70B    | 64GB–128GB   | 非常強但非常吃資源 |

## Chroma DB

> 是本地向量庫 SDK，一個專門儲存「向量資料」的資料庫，用在 AI 的搜尋、RAG（檢索增強生成）裡

### 作用於

1. 儲存向量
2. 查詢相似度
3. 把相似文件找出來給 LLM 用

### 向量資料

> 一段文字 → 經過 embedding 模型 → 變成像 [0.13, -0.82, 0.56, …] 的數字列表，這些數字叫「向量」，代表文字的語意。  
> 向量就像文字的「座標」，AI 對比座標距離就知道哪段文字比較相似。

### 優點

- Local-first：可以離線跑，不會外傳資料（你非常需要）
- 安裝超簡單：不像 pinecone、weaviate 要伺服器
- 有 Node.js + Python（你前端工程師更方便用 Node）
- 速度快，適合本地 + 小中規模 RAG

### 使用方式

| 你操作                | 說明              |
| --------------------- | ----------------- |
| npm install chromadb  | 安裝 Chroma       |
| new ChromaClient(...) | **= 啟動 Chroma** |
| getOrCreateCollection | **= 建立資料表**  |
| add / query           | **= 開始使用 DB** |

### 如何查看 DB 裡的資料

> Chroma 是 SQLite + Parquet

可以直接用 SQLite Viewer 打開(用 VSCode SQLite 插件就能看到資料)：  
例如：company-rag-node/chroma-db/chroma.sqlite3

## 語意搜尋

語意搜尋是「比意思」，不需要文字完全一樣也能找出相關內容。

> 核心在於：  
> ➡ 把文字轉成向量（Vector）  
> ➡ 比較向量的距離（通常是 cosine similarity）

## Jaccard（雅卡爾係數 / Jaccard Similarity）

用來衡量兩段文字「相似度」的數學方法

> 數值範圍是 0 ～ 1：
>
> > 1 → 幾乎一模一樣  
> > 0 → 完全沒交集

在語意搜尋扮演的角色:

1. 判斷兩個 chunk 的內容是不是「其實在講同一件事」
2. 用來做「去重 / 防噪音」的輔助工具

為什麼不用 embedding 距離來去重？

| 方法               | 適合做什麼       | 不適合                   |
| ------------------ | ---------------- | ------------------------ |
| embedding distance | 語意搜尋         | chunk 去重（慢、成本高） |
| **Jaccard**        | **文字重複檢測** | 深層語意判斷             |
